cpp核心编程(四)类和对象

类和对象

c++面向对象的三大特性为：封装、继承、多态
c++认为万事万物皆为对象，对象上尤其属性和行为

  1. 封装继承多态
  2. 对象

封装

  1. 讲属性和行为作为一个整体，表现为生活中的事物
  2. 将属性和行为加以权限控制

封装意义二设计学生类

封装的意义

类在设计时，可以把属性和行为放在不同的权限下，加以控制
访问权限有三种

  1. public 公共权限
  2. protected 保护权限
  3. private 私有权限

struct 与 class 的区别

  1. struct的默认权限时public
  2. class的默认权限是私有private

成员属性设置为私有

优点

  1. 将所有成员属性设置为私有，可以自由控制读写权限
  2. 对于写权限，我们可以检测数据的有效性

封装案例

1.立方体类

  1. 建立立方体类别（cube）
  2. 求出立方体的面积和体积
  3. 分别用全局函数和成员函数判断两个立方体是否相等

2.点和圆的关系

设计一个园形类（circle）和一个点类（point），计算点和圆的关系

4.2对象的初始化和清理

生活中我们买的电子产品都会有出场设置，在某天不适用的时候会删除一些信息数据保障安全
c++中面向对象来源于生活，每个对象也都会有初始设置以及清理数据的设置

4.2.1 构造函数和析构函数

对象的初始化和强力也是非常重要的安全问题
一个对象或者变量没有初始状态对其使用的后果是为止，同样的使用完一个对象或者变量，没有及时的清理，也会造成一定的安全问题
c++利用了构造函数和析构函数解决上述问题，两个函数将会被编译器自动调用，完成对函数的初始化和清理工作。
对象的初始化和清理工作是编译器要求我们所做的事情，如果我们不提供构造和析构，编译器会提供
编译器提供的构造和析构函数是空实现

  1. 构造函数，主要作用是创建对象时为对象提供成员属性赋值，构造函数是由编译器自动调用，无需手动调用
  2. 析构函数，主要作用是在于对象销毁前系统自动调用，执行一些清理工作。

构造函数函数语法：类名（）{}

  1. 构造函数，没有返回值也不写void
  2. 函数名称与类名相同
  3. 构造函数可以有参数，因此可以发生重载
  4. 程序在调用对象的时候慧自动调用结构，无需手动调用，且只会调用一次

析构函数函数语法：~类名(){}

  1. 析构函数没有返回值，可以不写void
  2. 函数名称与类名相同，在名称前加上符号~~
  3. 析构函数不可以有参数，因此不可以发生重载
  4. 程序在对象销毁前会自动沿用析构，无需手动调用，而且只会调用一次。

构造函数的分类及调用

  1. 两种分类方式：
    按参数分为：有参构造和无参构造
    按类型分为：普通构造和拷贝构造
  2. 三种调用方式：括号法，显示法，隐式转换法
    两个注释，注释一构造函数匿名初始化，构造函数，析构函数自动调用后系统回收匿名对象
    注释二不要在匿名初始化拷贝构造函数

拷贝函数调用时机

c++中拷贝函数调用时机通常有三种情况

  1. 使用一个已经创建完毕的对象来初始化一个新的对象
  2. 值传递的方式给函数参数赋值
  3. 以值的方式返回局部对象

构造函数的调用规则

默认情况下，c++编译器至少给一个类添加3个函数

  1. 默认构造函数（无参，函数体为空）
  2. 默认析构函数（无参，函数体为空）
  3. 默认拷贝函数构造函数，对属性进行值拷贝

构造函数调用规则如下：
如果用户定义有构造函数，c++不再提供默认无参构造，但是会提供默认拷贝构造函数
如果用户定义了拷贝构造函数，则不会提供其他构造函数

深拷贝浅拷贝问题

深浅拷贝是面试的一个经典的问题。也是常见的一个坑
浅拷贝：简单的赋值拷贝操作
深拷贝：在堆区重新申请空间，进行拷贝操作
